## 测试概述

| 项目 | 说明 |
|------|------|
| 测试对象 | phonefast daemon/direct 模式全功能 |
| 测试框架 | Python 3 脚本 `test_install_cases.py` |
| 测试范围 | 18 个测试用例，覆盖 daemon、observe、tap、swipe、key、wait、包检查、错误处理 |
| 模式说明 | 自 v1.x 起**默认走 daemon 模式**；用 `--foreground` 切换 direct 模式。`--daemon` 标志保留兼容（等同默认） |
| 运行前提 | ADB 设备已连接 + phonefast 二进制已编译 + ffmpeg 可用 |
| 预期耗时 | 快速模式 ~30s，完整模式 ~90s |

### 设计原则

1. **每条测试独立可运行**: 不依赖特定页面状态（除 TC-9/TC-11）
2. **正反验证**: 正确路径和错误路径都覆盖
3. **逻辑正确性优先**: 先确保命令返回正确，再验证设备行为
4. **防御性断言**: 遇到预期外的页面状态时跳过而非报错

## 关键踩坑记录与验证

### 踩坑 1: `adb pm path` 缺少 `shell`

| | |
|---|---|
| **错误用法** | `adb pm path <pkg>` |
| **正确用法** | `adb shell pm path <pkg>` |
| **后果** | 全部误报为未安装（返回码始终非零） |
| **TC 覆盖** | TC-8.1, TC-8.2 |
| **预防** | 永远不要使用 `adb pm path` |

### 踩坑 2: `tap_element` 不稳定性

| | |
|---|---|
| **问题** | `phonefast --daemon tap_element text='安装'` 返回 element not found |
| **原因** | Play 商店 Compose UI 的元素 text/desc 可能与预期不符 |
| **解决** | 优先用 `observe` 获取 bounds → 计算坐标 → 用 `tap <x> <y>` |
| **TC 覆盖** | TC-12, TC-10 |

### 踩坑 3: Play 商店安装按钮不响应快速 tap

| | |
|---|---|
| **问题** | 10ms DOWN→UP 间隔被 Play 商店反自动化校验拒绝 |
| **原因** | phonefast `Tap()` 原硬编码 10ms，人类最短触摸 ~50ms |
| **修复** | `session.go:TapDelay` 默认 10ms → 应 ≥ 50ms |
| **TC 覆盖** | TC-4（验证 tap 命令本身语法正确）；实际按钮需 ≥50ms 延迟 |
| **相关文档** | `phonefast-tap-fix.md` |

### 踩坑 4: Daemon 偶尔无响应

| | |
|---|---|
| **现象** | daemon 模式命令偶尔返回错误 |
| **原因** | 视频/控制连接断开（TCP keepalive 15-30s） |
| **恢复** | healthLoop (10s 周期) 自动重连 |
| **TC 覆盖** | TC-15 |

### 踩坑 5: `python` vs `python3`

| | |
|---|---|
| **问题** | macOS 默认 `python` 可能不存在 |
| **解决** | 始终使用 `python3` |
| **TC 覆盖** | 脚本 shebang 使用 `#!/usr/bin/env python3` |

## 运行测试

### 快速运行

```bash
# 快速模式（跳过 Play 商店导航和锁屏测试）
python3 tests/test_install_cases.py --quick

# 完整测试
python3 tests/test_install_cases.py

# 跳过 daemon 重启测试（保持当前 daemon 状态）
python3 tests/test_install_cases.py --skip-daemon-tests
```

### 输出格式

```bash
# 标准输出（人类可读）
python3 tests/test_install_cases.py

# JSON 输出（CI 集成）
python3 tests/test_install_cases.py --json-out

# JSON 报告写入文件
python3 tests/test_install_cases.py --report /tmp/phonefast_test_report.json
```

### 与现有测试的关系

| 现有测试 | 本测试补充 | 关系 |
|----------|-----------|------|
| `test_mcp.py` | MCP 协议层面 | **互补**: MCP 测协议格式，本测试测业务逻辑 |
| `test_e2e.py` | 原始 scrcpy socket | **互补**: e2e 测连接建立，本测试测操作语义 |
| `test_release.py` | help/devices/screenshot | **增强**: 本测试增加状态转换和错误路径 |
| `benchmark.py` | 延迟分布 | **互补**: benchmark 测性能，本测试测正确性 |

```bash

bash scripts/build.sh

# 2. 运行单元测试
```

## 准入标准

| 级别 | 标准 | 条件 |
|------|------|------|
| **必须通过** | TC-1 ~ TC-8, TC-10, TC-12, TC-14, TC-16, TC-17 | 基础命令正确性 |
| **应该通过** | TC-9, TC-11, TC-13, TC-15, TC-18 | 依赖设备/页面状态 |
| **可选** | TC-4.2 (direct 模式) | daemon 运行中 direct 可能冲突 |

### 失败分类

| 类别 | 示例 | 严重度 |
|------|------|--------|
| **二进制缺失** | binary not found | FATAL — 测试不执行 |
| **设备未连接** | no devices | FATAL — 测试不执行 |
| **命令返回错误** | tap 返回非零退出码 | HIGH — phonefast 逻辑缺陷 |
| **预期行为不符** | observe 缺少必填字段 | HIGH — 协议不兼容 |
| **超时** | daemon 启动超时 | MEDIUM — 设备/环境问题 |
| **页面状态不符** | 元素文本缺失 | LOW — 非 phonefast 问题 |

### 修复后验证清单

当修复 phonefast 代码后，按以下顺序验证：

1. `go test ./...` — 单元测试
2. `python3 tests/test_install_cases.py --quick` — 逻辑正确性快速检查
3. `python3 tests/test_install_cases.py` — 完整业务流测试
4. `python3 tests/test_mcp.py` — MCP 协议兼容性
5. `bash tests/gate.sh` — 一站式准出
